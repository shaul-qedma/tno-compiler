# Plot deck

Eight plots summarizing what we've learned about the MPO-based
brickwall compiler. Organized into four acts; each plot is
self-contained with a plain-language annotation box on the figure.

Generated by `scripts/make_plots.py` from the CSVs in `docs/data/`.

## Act 1 — What the compiler does

- **01_min_depth_vs_steps.png** — Minimum compiled depth (for < 5%
  worst-case error) vs Trotter steps, per target difficulty.
  The naive rule `⌈steps/2⌉` is conservative: gapped targets
  compress to depth 3 even at 16 Trotter steps.
- **02_n_scaling.png** — Accuracy vs number of qubits at fixed
  (difficulty, evolution length, compiled depth). Error grows
  only 1.5–2× from n=4 to n=12.
- **03_landscape.png** — Three-panel heatmap over (Trotter steps,
  compiled depth), one panel per coupling regime. Full landscape
  view.

## Act 2 — Frobenius is a reliable channel-distance proxy

- **04_frobenius_vs_td.png** — Scatter across 24 circuits spanning
  six orders of magnitude in quality. Worst-case channel error is
  within ~20% of the cheap matrix-distance calculation the
  compiler optimizes.
- **05_max_mean_ratio.png** — Violin: (worst-case error / typical
  error) ratio per circuit, binned by quality. Always between
  1.0 and 1.2; actually shrinks for worse compiles.

## Act 3 — Ensemble QP is best-selection, not cooperative

- **06_ensemble_vs_single.png** — Three-bar chart per test config:
  worst single, best single, weighted ensemble. Ensemble tracks
  best single within 1-3%, never meaningfully beats it. The only
  real value is routing mass away from bad basins (red bars,
  10–250× worse).

## Act 4 — Add-on mechanisms don't move the needle

- **07_seed_shoulder.png** — Seed disagreement peaks at the
  minimum required depth (d=4–5). Below or above: all random
  starts find the same basin, no ensemble benefit possible.
- **08_mechanism_benefits.png** — Strip chart of per-config
  improvements for dropout, paired perturbation (Kalloor's recipe),
  and paired perturbation with over-tight circuits. Mostly gray
  (no change), occasionally green (meaningful), never
  reliably substantial.

## Key quantitative findings

- Normalized Frobenius `‖U − V‖_F / √d` ≈ (mean trace distance) to
  within 11%; ≈ (worst-case trace distance) / 1.06 to 1.22.
- Ensemble QP beats best single by ≤3.5% across all tested configs.
- Worst single is 9–281× worse than best single (basin lottery is real).
- Every add-on mechanism tested tops out at ~10-20% improvement on
  specific cases, with ~50% showing essentially no benefit.
